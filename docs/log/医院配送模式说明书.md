# 医院配送模式说明书

## 1. 业务目标

医院配送模式用于机器人在医院场景下执行一轮完整的器械配送流程，核心链路为：

1. 从首页进入医院配送模式。
2. 在医院配送页为货层配置物品和房间。
3. 启动任务后先去消毒间。
4. 人工确认后继续前往各个房间。
5. 到房间后开门、取货、确认收货。
6. 全部完成后自动返航到原点。
7. 过程结果写入医院配送历史记录。

### 2.2 医院配送配置页

页面文件：[HospitalDeliveryActivity.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/ui/main/HospitalDeliveryActivity.java#L46)

该页面负责“任务配置”和“任务启动前校验”。

- 初始化时调用 `HospitalDeliveryManager.init()`，确保历史记录可读。
- 从 `ServiceManager` 获取：
  - `IDoorService`，用于开关门、查询门状态
  - `IRobotStateService`，用于定位和获取地图点位
- 通过 `getDestinationList()` 拉取地图点位后，按类型拆分为：
  - `originPoints`：原点
  - `disinfectionPoints`：消毒间
  - 普通房间点位列表
- 支持 3 个货层 `L1/L2/L3`，每层需要同时配置：
  - 一个物品预设
  - 一个房间点位
- 启动前校验：
  - 至少一个货层完成“物品 + 房间”配对
  - 地图中必须存在原点
  - 地图中必须存在消毒间
  - 如有 `robotStateService`，会先执行定位校验
  - 舱门未关闭时，先自动关门，延迟 5 秒再启动

### 2.5 房间到达与收货确认

到达房间后，导航页不会自己完成收货，而是跳转到统一签收页：

- [ConfirmReceiptActivity.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/ui/main/ConfirmReceiptActivity.java#L50)

导航页触发点：

- [HospitalDeliveryNavigationActivity.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/ui/main/HospitalDeliveryNavigationActivity.java#L561)

签收页业务逻辑：

- 根据 `pairings` 和当前房间 `current_node`，计算本房间要取走的货层并高亮显示
- 默认启动 30 秒离场倒计时
- 第一次点击按钮时先验密，再执行开门
- 开门成功后，按钮切换为“确认收货”
- 确认收货时关闭舱门，确认门真正关闭后退出页面
- 如果超时未操作，则按超时逻辑自动离场
- 如果是医院配送模式，记录写入 `HospitalDeliveryManager`，阶段固定为 `STAGE_ROOM`

## 3. 异常与中断逻辑

### 3.1 启动前异常

- 未配置货层/物品/房间，不允许启动
- 无原点或无消毒间，不允许启动
- 定位失败，跳转 `PositioningFailedActivity`
- 自动关门失败，停止启动并提示

对应代码：

- [HospitalDeliveryActivity.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/ui/main/HospitalDeliveryActivity.java#L129)

### 3.2 导航异常

- 导航准备失败、设速失败、设目标失败时，直接记录导航失败并退出当前页
- 运行中收到 `onNavigationError()` 时，也会按当前阶段记录导航失败

对应代码：

- 导航准备异常：[HospitalDeliveryNavigationActivity.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/ui/main/HospitalDeliveryNavigationActivity.java#L293)

### 3.3 人工结束/返航

- 导航页双击屏幕可暂停
- 暂停后可：
  - 结束任务
  - 开关舱门
  - 直接返航
  - 继续配送
- 暂停 30 秒超时会自动继续
- 如果在消毒间阶段或房间阶段中途返航，会先记录当前阶段为 `STATUS_CANCELLED`

### 4.2 记录模型

记录模型：

- [HospitalDeliveryRecord.java](/d:/android_work/WeigaoRobotControl/app/src/main/java/com/weigao/robot/control/model/HospitalDeliveryRecord.java#L9)

字段说明：

- `taskId`：一次医院配送任务唯一标识
- `startTime`：任务开始时间
- `endTime`：当前阶段完成时间
- `pointName`：点位名称
- `duration` / `formattedDuration`：从任务开始到当前阶段的累计耗时
- `status`：阶段状态
- `stage`：阶段类型

状态定义：

- `STATUS_SUCCESS`
- `STATUS_FAILED_TIMEOUT`
- `STATUS_FAILED_HARDWARE`
- `STATUS_NAV_FAILED`
- `STATUS_CANCELLED`

阶段定义：

- `STAGE_DISINFECTION`
- `STAGE_ROOM`
- `STAGE_RETURN`

注意：

- 当前耗时是“从整单开始到当前阶段完成”的累计耗时，不是单段耗时。

## 7. 当前实现特征总结

- 医院配送是建立在现有物品配送能力上的一条定制流程。
- 与普通物品配送相比，新增了“先到消毒间，再去房间”的阶段控制。
- 收货页复用了 `ConfirmReceiptActivity`，通过 `record_mode = hospital` 切换为医院配送记录。
- 返航页复用了通用 `ReturnActivity`。
- 历史记录单独使用 `HospitalDeliveryManager` 和 `hospital_delivery_history.json` 管理。
